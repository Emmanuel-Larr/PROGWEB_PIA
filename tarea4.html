    Compilador


    Un compilador principalmente es un programa informático que traduce un programa escrito en un lenguaje de programación, definido «código fuente», a otro lenguaje de alto nivel tales como COBOL, PASCAL, BASIC, C, etc. ademas de un de medio o más bajo nivel código que seria intermedio o código máquina como puede ser C/C++.

    Ademas que un compilador generalmente genera lenguaje ensamblador primero y luego traduce el lenguaje ensamblador al lenguaje máquina. Una utilidad conocida como "enlazador" combina todos los módulos de lenguaje de máquina necesarios en un programa ejecutable que se puede ejecutar en la computadora.



    Origen


    Su origen se remonta al primer compilador que fue escrito por Grace Hopper, en 1952 para el lenguaje de programación A-0. Ademas que en 1950 John Backus dirigió una investigación en IBM sobre un lenguaje algebraico. Gracias a estos 2 pioneros de la programacion surgió por primera vez el concepto de un traductor como un programa que traducía un lenguaje a otro lenguaje, que luego seria conocido como el compilador, ya despues de unos años de avances tecnologicos en el campo de la programacion.

    Ademas que en 1954 se empezó a desarrollar un lenguaje que permitía escribir fórmulas matemáticas de manera traducible por un ordenador; le llamaron FORTRAN "FORmulae TRANslator". Este fue el primer lenguaje de alto nivel y se introdujo en 1957 para el uso de la computadora IBM modelo 704. Luego mas tarde En el caso particular de que el lenguaje a traducir es un lenguaje de alto nivel y el lenguaje traducido de bajo nivel, se emplea el término compilador.



    Tipos


    • Compilador cruzado: El compilador cruzado principalmente genera un código ejecutable en un ordenador distinto de aquel en que se realiza la compilación

    • Compilador de montaje y ejecución: Este compilador se encarga de fragmentar el programa fuente en módulos que se compilan por separado, y una vez compilados se unen mediante un enlazador para formar un módulo ejecutable.

    • Compilador en una pasada: Este compilador se encarga de examina el código fuente una sola vez, generando el código objeto.

    • Compilador de pasadas múltiples: Este compilador requiere de varias lecturas del programa fuente para producir y optimizar el código objeto.

    • Compilador de optimización: Este compilador principalmente lee el código fuente, lo analiza, optimiza y descubre errores potenciales sin ejecutar el programa.

    • Compilador incremental: El compilador incremental es el programa fuente, que en caso de detectar errores al volver a compilar el programa corregido, solo compila las modificaciones que se han hecho respecto al primero.

    • Ensamblador: El lenguaje fuente es principalmente el lenguaje ensamblador.

    • Autocompilador: Es el compilador que está escrito en el mismo lenguaje a compilar, básicamente nos sirve para hacer ampliaciones al lenguaje, mejorar el código generado, etc.

    • Metacompilador: Este se considera el compilador de compiladores, ya que este obtiene como entrada la definición de un lenguaje y como salida el compilador para dicho lenguaje.



    Interprete


    Un intérprete principalmente es un programa informático que procesa el código fuente de un proyecto de software durante su tiempo de ejecución, es decir, mientras el software se está ejecutando, y este actúa como una interfaz entre ese proyecto y el procesador. Ademas que un intérprete siempre suele procesar el código línea por línea, de modo que lee, analiza y prepara cada secuencia de forma consecutiva para el procesador.

    Pero en si, un intérprete o interpretador es un programa informático capaz de analizar y ejecutar otros programas, escritos en un lenguaje de alto nivel.



    Origen


    Como origen inicialmente, los lenguajes interpretados eran compilados línea por línea, es decir, cada línea era compilada a medida que estaba a punto de ser ejecutada, y si un bucle o una subrutina hicieran que ciertas líneas se ejecutaran múltiples veces, serían recompiladas repetidamente. Esto ha llegado a ser mucho menos común. La mayoría de los lenguajes interpretados usan una representación intermedia, que combina tanto la compilación como la interpretación.

    En este caso, un compilador puede producir el código byte o el código enhebrado, que entonces es ejecutado por un intérprete de código byte. Podemos tomar como ejemplo , Smalltalk de 1980, que fue diseñado para ser interpretado en tiempo de ejecución, permite a objetos genéricos interactuar dinámicamente entre sí.

A diferencia de otros lenguajes como java o .NET, se trata de un lenguaje interpretado, es decir, que no es necesario compilarlo para ejecutar las aplicaciones escritas en Python, sino que se ejecutan directamente por el ordenador utilizando un programa denominado interpretados, por lo que no es necesario “traducirlo” a lenguaje máquina.

El lenguaje Perl se percibe habitualmente como un lenguaje intermedio entre los shell scripts y la programación en C.
